# Контроль доступу

Система використовує дворівневий контроль доступу: **ролі (RBAC)** та **права доступу (Scopes/Permissions)**.

## Ролі користувачів

Роль визначає загальний рівень повноважень у системі.

| Роль      | Значення           | Опис                                                                 |
| --------- | ------------------ | -------------------------------------------------------------------- |
| `admin`   | `UserRole.ADMIN`   | Повний доступ до всіх модулів, налаштувань та логів аудиту           |
| `manager` | `UserRole.MANAGER` | Управління звітами, перегляд статистики та базове адміністрування    |
| `support` | `UserRole.SUPPORT` | Операційна робота: обробка заявок, керування IP-адресами та технікою |
| `client`  | `UserRole.CLIENT`  | Тільки перегляд доступних ресурсів (без права редагування)           |

::: tip
Роль `admin` автоматично обходить усі перевірки ролей та прав доступу.
:::

### Використання у резолверах

```typescript
@UseGuards(AccessTokenGuard, UserRoleGuard)
@Role(UserRole.MANAGER, UserRole.ADMIN)
@Query(() => [UserEntity])
async findAllUsers() { ... }
```

## Права доступу (Scopes)

Права доступу — це детальні дозволи у форматі `ресурс:дія`. Вони призначаються індивідуально кожному користувачу і дозволяють тонко налаштовувати можливості без зміни ролі.

### Формат

```
{ресурс}:{дія}
```

Доступні дії: `create`, `read`, `update`, `delete`, `notice`

### Повний список прав доступу

#### Events (Календар подій)

| Право          | Опис                 |
| -------------- | -------------------- |
| `event:create` | Створення подій      |
| `event:read`   | Перегляд подій       |
| `event:update` | Редагування подій    |
| `event:delete` | Видалення подій      |
| `event:notice` | Сповіщення про події |

#### Channels (Мережеві канали)

| Право            | Опис                  |
| ---------------- | --------------------- |
| `channel:create` | Створення каналів     |
| `channel:read`   | Перегляд каналів      |
| `channel:update` | Редагування каналів   |
| `channel:delete` | Видалення каналів     |
| `channel:notice` | Сповіщення про канали |

#### IP Addresses

| `ipaddress:delete` | Видалення IP-адрес       |

| `mailbox:delete` | Видалення скриньок      |
| `mailbox:notice` | Сповіщення про скриньки |

#### Requests (Заявки)

| Право            | Опис                  |

### Використання у резолверах

```typescript
@UseGuards(AccessTokenGuard, UserScopeGuard)
@Scope(UserScope.IPADDRESS_READ)
@Query(() => PaginatedIpaddress)
async findAllIpaddresses() { ... }
```

## Guards

| Guard               | Призначення                            |
| ------------------- | -------------------------------------- |
| `AccessTokenGuard`  | Перевірка валідності JWT access token  |
| `RefreshTokenGuard` | Перевірка валідності JWT refresh token |
| `UserRoleGuard`     | Перевірка ролі користувача             |
| `UserScopeGuard`    | Перевірка прав доступу (scope)         |
| `UserStatusGuard`   | Перевірка статусу облікового запису    |

## Помилки авторизації

| HTTP-код | GraphQL Extensions | Причина                       |
| -------- | ------------------ | ----------------------------- |
| 401      | `UNAUTHORIZED`     | Токен відсутній або недійсний |
| 403      | `FORBIDDEN`        | Недостатньо прав або ролі     |
| 403      | `FORBIDDEN`        | Обліковий запис не активний   |
