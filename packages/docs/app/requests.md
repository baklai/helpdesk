# Сервісна підтримка (Requests)

Модуль управления заявками пользователей (Helpdesk), обеспечивающий полный цикл обработки инцидентов.

## Структура заявки (Request Object)

| Поле          | Описание                                      |
| :------------ | :-------------------------------------------- |
| `reqnum`      | Уникальный инвентарный номер заявки           |
| `requestor`   | ФИО и контактные данные заявителя             |
| `description` | Суть проблемы или запроса                     |
| `conclusion`  | Итоговое решение (заполняется при закрытии)   |
| `status`      | Текущее состояние (Open, Closed, In Progress) |

## Процесс работы

1. **Создание**: Пользователь или оператор создает заявку через мутацию `CREATE_ONE_REQUEST`.
2. **Назначение**: Система фиксирует `openedBy` (сотрудник, принявший заявку).
3. **Выполнение**: После решения проблемы заполняется поле `conclusion`.
4. **Закрытие**: Фиксируется `closedBy` и дата закрытия.

## Связанные данные

Заявки автоматически связываются с объектами `Department` и `Location` для формирования аналитической отчетности по подразделениям.
